# Contributing guide

## Branching

- Create a branch from `main` for each change.
- Keep pull requests focused and small.

## Commits

- Use clear, imperative commit messages.
- Example: `Add backend API starter docs`.

## Pull requests

Please include:

- What changed.
- Why it changed.
- How it was tested.
- Any follow-up work.

## Documentation standard

If behavior changes, update relevant README/docs in the same PR.
